**Description:**

As an HR administrator, I need a comprehensive employee management screen where I can view all employees in a clear layout, add new employees with complete information, edit existing employee records, and manage employee data efficiently.

**User Stories:**

**1. View Employee List (Admin/Manager)**
As an admin or manager, I want to see all employees in a grid/card layout so that I can quickly browse through the team and access employee information.

**2. Add New Employee**
As an admin, I want to add a new employee with all their details in a single form so that I can onboard new team members efficiently.

**3. Edit Employee Information**
As an admin, I want to edit any employee's information so that I can keep records up-to-date when details change.

**4. View My Profile (Employee)**
As an employee, I want to view and update my own profile information so that I can ensure my personal details are accurate.

**5. Delete Employee Record**
As an admin, I want to delete employee records so that I can remove terminated employees from the system.

**Acceptance Criteria:**

**Functional Requirements:**

**1. Page Layout**
- [ ] Page title displays "Employees" for admin/manager, "My Profile" for employee
- [ ] Header contains page title and action buttons
- [ ] Export to Excel button visible to all users
- [ ] Add Employee button visible only to admin/manager
- [ ] Responsive layout that works on mobile, tablet, desktop

**2. Employee List View (Admin/Manager)**
- [ ] Display employees in a grid layout (3 columns on desktop, 2 on tablet, 1 on mobile)
- [ ] Each employee card shows:
  - [ ] Title (Mr, Mrs, Dr, etc.) + Full Name
  - [ ] Job Title
  - [ ] Email address
  - [ ] Department
  - [ ] NI Number (if available)
  - [ ] "Director" badge for DIRECTOR employee type
  - [ ] Bank Details section (if available)
  - [ ] Emergency Contact section (if available)
- [ ] Edit and Delete buttons on each card
- [ ] Cards have hover effect for better UX

**3. Employee Profile View (Employee Role)**
- [ ] Display single card with full employee details
- [ ] Sections organized by:
  - [ ] Personal Information (email, phone, department, NI number, start date)
  - [ ] Bank Details (bank name, masked account number, sort code)
  - [ ] Emergency Contact (name, phone, relationship, address)
- [ ] "Update Profile" button to edit own information
- [ ] Clean, readable layout with proper spacing
- [ ] Masked sensitive data (account number shows last 4 digits only)

**4. Add/Edit Employee Form**
- [ ] Form opens in expandable section above employee list
- [ ] Form title shows "New Employee" or "Edit Employee"
- [ ] All form fields with proper labels and placeholders:

**Personal Details:**
- [ ] First Name * (required, text input)
- [ ] Last Name * (required, text input)
- [ ] Title (optional, text input - "Mr, Mrs, Dr, etc.")
- [ ] Email * (required, email validation)
- [ ] Phone Number (optional, text input)
- [ ] Job Title * (required, text input)
- [ ] Employee Type (dropdown: Employee/Director)
- [ ] Department (optional, text input)
- [ ] NI Number (UK format, optional)
- [ ] Employment Start Date (date picker with label)

**Bank Details Section:**
- [ ] Section header with divider
- [ ] Bank Name (optional, text input)
- [ ] Account Number (optional, text input)
- [ ] Sort Code (optional, text input)

**Emergency Contact Section:**
- [ ] Section header with divider
- [ ] Emergency Contact Name (optional, text input)
- [ ] Emergency Contact Phone (optional, text input)
- [ ] Relationship (optional, text input)
- [ ] Emergency Contact Address (optional, full-width text input)

**Form Actions:**
- [ ] Submit button shows "Add Employee" or "Update Employee"
- [ ] Cancel button (visible only when editing)
- [ ] Form clears on successful submission
- [ ] Success message after add/update
- [ ] Error message if submission fails

**5. Form Validation**
- [ ] Required fields show * indicator
- [ ] Email field validates email format
- [ ] Date picker for Employment Start Date
- [ ] All fields trim whitespace
- [ ] Form cannot submit without required fields
- [ ] Clear error messages for validation failures

**6. Export to Excel**
- [ ] Button triggers download of Excel file
- [ ] File named: "employees-YYYY-MM-DD.xlsx"
- [ ] Includes all employee data
- [ ] Success/error feedback to user

**7. Delete Confirmation**
- [ ] Confirmation dialog before deletion
- [ ] "Are you sure you want to delete this employee?"
- [ ] Requires explicit confirmation
- [ ] Success message after deletion
- [ ] Employee list refreshes automatically

**8. Role-Based Access Control**
- [ ] Admin/Manager see all employees
- [ ] Employee sees only their own profile
- [ ] Add/Edit/Delete buttons hidden for employee role
- [ ] Export available to all roles

**Non-Functional Requirements:**

**1. Performance**
- [ ] Page loads in <2 seconds
- [ ] Form submission completes in <1 second
- [ ] Smooth transitions and animations
- [ ] No UI freezing during operations

**2. Usability**
- [ ] Intuitive form layout with logical field grouping
- [ ] Clear visual hierarchy
- [ ] Consistent button styling and placement
- [ ] Helpful placeholder text in all fields
- [ ] Date picker for date fields (not manual entry)
- [ ] Form remembers data when switching between add/edit

**3. Accessibility**
- [ ] Proper form labels for screen readers
- [ ] Keyboard navigation support
- [ ] Focus states on interactive elements
- [ ] Sufficient color contrast for text
- [ ] ARIA labels where appropriate

**4. Responsive Design**
- [ ] Grid adapts: 3 cols (desktop) → 2 cols (tablet) → 1 col (mobile)
- [ ] Form fields stack properly on mobile
- [ ] Buttons remain accessible on small screens
- [ ] Touch-friendly targets (min 44x44px)
- [ ] Horizontal scrolling prevented

**5. Dark Mode Support**
- [ ] All components support dark mode
- [ ] Proper contrast in both light/dark themes
- [ ] Form inputs styled for dark mode
- [ ] Cards and borders visible in dark mode
- [ ] Text remains readable in both themes

**Technical Requirements:**

**1. Frontend Stack**
- React with TypeScript
- Tailwind CSS for styling
- React Icons (HiPlus for Add button)
- Card component from shared components

**2. API Integration**
- GET /employees - Load all employees
- POST /employees - Create new employee
- PUT /employees/:id - Update employee
- DELETE /employees/:id - Delete employee
- GET /employees/export/excel - Export to Excel

**3. State Management**
- useState for form data, items list, UI states
- useEffect for initial data load and role detection
- useMemo for filtered items based on role

**4. Form State**
```typescript
{
  firstName: string
  lastName: string
  title: string
  email: string
  phoneNumber: string
  jobTitle: string
  employeeType: 'EMPLOYEE' | 'DIRECTOR'
  department: string
  niNumber: string
  startDate: string (YYYY-MM-DD)
  bankName: string
  accountNumber: string
  sortCode: string
  emergencyContactName: string
  emergencyContactPhone: string
  emergencyContactRelation: string
  emergencyContactAddress: string
}
```

**5. JWT Token Parsing**
- Parse JWT to extract user role (ADMIN/MANAGER/USER)
- Parse JWT to extract user email
- Use role for conditional rendering
- Use email to filter employee data for USER role

**Dependencies:**
- Employee API endpoints (backend)
- Authentication middleware (JWT)
- Shared Card component
- API helper functions (apiGet, apiPost, apiPut, apiDelete)

**Risks:**
- Form complexity may confuse users
- Large number of fields could feel overwhelming
- Mobile form usability with many fields
- Sensitive data (bank details) display security
- Performance with large employee datasets
